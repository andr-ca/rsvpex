## Form Service Selection

### Decision: Web3Forms (Free Tier)

### Rationale:
- **Generous free tier**: 250 submissions/month sufficient for pre-launch campaign
- **Pure HTML integration**: Works with static HTML forms, no JavaScript required for basic functionality
- **Simple setup**: Just add access key and form action URL
- **Built-in spam protection**: Free tier includes honeypot fields and hCaptcha integration
- **Privacy-focused & GDPR compliant**: Does not store submissions on servers, sends directly to email
- **Progressive enhancement friendly**: Works with basic HTML, optional JavaScript enhancements
- **No vendor lock-in**: Submissions go directly to email inbox

### Alternatives Considered:
- **Formspree**: Only 50 submissions/month on free tier - too restrictive for pre-launch. Paid plan required ($10/month).
- **Netlify Forms**: 100 submissions/month free, but tightly coupled to Netlify hosting platform. Not platform-agnostic.
- **Basin**: Only 50 submissions/month with 30-day data retention. Good analytics but low submission limits.
- **Getform**: Restrictive rate limiting (1 submission per minute), not suitable for traffic spikes.

### Integration Approach:

```html
<form action="https://api.web3forms.com/submit" method="POST">
  <input type="hidden" name="access_key" value="YOUR_ACCESS_KEY">
  <input type="email" name="email" placeholder="Your Email" required>
  <input type="checkbox" name="botcheck" class="hidden" style="display: none;">
  <input type="hidden" name="redirect" value="https://yoursite.com/thank-you.html">
  <input type="hidden" name="subject" value="New Beta Sign-up">
  <button type="submit">Join the Waitlist</button>
</form>
```

**Setup steps**:
1. Sign up at web3forms.com (free, no credit card)
2. Get access key from dashboard
3. Replace `YOUR_ACCESS_KEY` in form
4. Create `thank-you.html` for post-submission redirect
5. Hide honeypot field with CSS

**Environment variable**: `WEB3FORMS_ACCESS_KEY` (public-safe, acts as email alias)

## Email Collection & Deduplication Workflow

### Decision: Zero-Budget Workflow with Manual Deduplication + Mailchimp

### Email Collection Process:

**How emails arrive**:
1. Visitor submits form on landing page
2. Web3Forms sends email to your inbox with submission details
3. Email subject: "New Beta Sign-up"
4. Email body contains: email address, name (if collected), timestamp

**Collection workflow** (zero cost):

```
Landing Page Form
       ↓
   Web3Forms (free)
       ↓
Email arrives in: yourname@gmail.com
       ↓
Gmail Filter/Folder: "RSVPex Beta Signups"
  (Rule: from=noreply@web3forms.com → move to folder)
       ↓
Every week or at 50+ signups: Export emails
       ↓
Mailchimp (free tier, 500 contacts max)
  - Import CSV file
  - Auto-deduplicates (won't add same email twice)
  - Organizes subscriber list
       ↓
At Beta Launch: Send announcement via Mailchimp campaign
  - Can track open rates, click rates
  - Professional delivery vs BCC method
```

### Step-by-Step Implementation:

**Step 1: Gmail Filter (One-time Setup)**
1. In Gmail: Settings → Filters and Blocked Addresses → Create a new filter
2. From: `noreply@web3forms.com`
3. Action: Create label "RSVPex Beta Signups" and auto-archive (or skip inbox)
4. All Web3Forms emails now go to this folder automatically

**Step 2: Weekly Email Export**
1. Go to Gmail label: "RSVPex Beta Signups"
2. Select all emails
3. More → Download (creates .csv with email data)
4. Save to: `beta_signups_week_X.csv`

**Step 3: Mailchimp Setup (One-time, 5 minutes)**
1. Sign up: mailchimp.com (free tier)
2. Create audience: "RSVPex Beta Waitlist"
3. Get your Audience ID from settings (needed later if automating)

**Step 4: Import to Mailchimp (Weekly)**
1. Mailchimp dashboard → Audience → Manage contacts → Import
2. Upload the CSV file from Step 2
3. Map columns: Email → Email field
4. Mailchimp automatically deduplicates (won't add same email twice)
5. Check "Update existing contacts if found" checkbox

**Step 5: At Beta Launch - Send Email Campaign**
1. Mailchimp → Create Campaign → Email
2. Choose audience: "RSVPex Beta Waitlist"
3. Write launch announcement
4. Send → Mailchimp tracks opens, clicks, bounces
5. Export results afterward

### Deduplication Details:

**How deduplication works**:
- **Mailchimp automatic dedup**: If email `jane@example.com` already in list, importing again skips the duplicate
- **Manual method (if not using Mailchimp)**: Open CSV file → sort by email → delete rows with same email

**Example CSV structure** (auto-generated by Gmail export):
```
Email,Name,Date
jane@example.com,Jane Smith,2025-10-25
john@example.com,John Doe,2025-10-25
jane@example.com,Jane Smith,2025-10-26  ← Duplicate, Mailchimp won't add
```

### Cost Breakdown:

| Service | Free Tier | Cost |
|---------|-----------|------|
| Web3Forms | 250 submissions/month | $0 |
| Mailchimp | 500 contacts | $0 |
| Cloudflare Analytics | 100K requests/month | $0 |
| **Total** | **Sufficient for pre-launch** | **$0** |

### Future Scaling (If needed):

If you get >50 signups/week and want automation:
- **Upgrade Web3Forms to Pro**: $5/month
- Add webhook integration to automatically sync to Mailchimp in real-time
- Eliminates manual weekly export/import

**Recommendation**: Start with free manual workflow. Upgrade to Pro only if signup volume justifies the $5/month.

## Image & Visual Asset Strategy

### Decision: SVG Icons + Minimal Illustration (Hybrid Approach)

### Rationale:
- **Performance**: SVG icons 2-8KB each, hero illustration 15-30KB optimized = 50-80KB total (well under 100KB target)
- **Modern aesthetic**: Clean illustrations align with "Clean & Sleek Design" constitution principle
- **Scalability**: SVG graphics maintain crisp quality across all screen sizes (320px-2560px)
- **Zero cost**: unDraw (MIT License) + Heroicons (MIT License) - unlimited commercial use
- **Technical benefits**: Inline SVG enables CSS customization, no JavaScript required, perfect for vanilla-first approach

### Alternatives Considered:
- **Minimal graphics (icons only)**: Too sparse for coming-soon announcement; lacks emotional appeal for >10% conversion
- **Stock photos**: 80-150KB even optimized, generic aesthetic, requires WebP/AVIF conversion complexity
- **Hybrid with photo hero**: Pushes budget to 150-200KB, risks missing <2s load time target

### Implementation:

**Formats**:
- **Icons**: Inline SVG (2-8KB each, 3-5 icons total)
- **Hero illustration**: Inline SVG or external SVG (15-30KB optimized)
- **Favicon**: SVG + PNG fallbacks (16x16, 32x32, 180x180 for Apple)

**Sources**:
- **Icons**: Heroicons (https://heroicons.com) - 1,288 MIT-licensed icons, clean modern style
- **Hero illustration**: unDraw (https://undraw.co) - 1,000+ customizable illustrations, search "events", "calendar", "planning"

**Optimization tools**:
- **SVGOMG** (https://jakearchibald.github.io/svgomg/) - 30-50% file size reduction without quality loss
- Enable gzip/Brotli on hosting: 30KB SVG → 6-9KB transferred

**Performance budget**:
| Asset Type | Quantity | Size Each | Total |
|------------|----------|-----------|-------|
| Hero illustration (SVG) | 1 | 25KB | 25KB |
| Benefit icons (SVG) | 4-5 | 3KB | 12-15KB |
| Favicon set | 3 | 3KB | 9KB |
| **Total** | | | **46-49KB** |
| **After gzip** | | | **~12-15KB transferred** |

**Accessibility**:
```html
<!-- Decorative icons -->
<svg aria-hidden="true" focusable="false">...</svg>

<!-- Meaningful illustration -->
<svg role="img" aria-labelledby="hero-title">
  <title id="hero-title">Team collaborating on event planning</title>
</svg>
```

## Font Loading Strategy

### Decision: System font stack with optional self-hosted variable font for headings

### Rationale:
- **Performance**: System fonts deliver zero latency (no HTTP requests, no DNS lookups, no layout shifts)
- **15-22% Lighthouse improvement**: Removing web fonts provides immediate performance boost
- **Professional appearance**: Modern system fonts (Inter on Linux, SF Pro on macOS, Segoe UI on Windows) used by major tech companies
- **Implementation simplicity**: Pure CSS solution, no build tools, no JavaScript

### Alternatives Considered:
- **Google Fonts**: Adds 100-300ms latency minimum despite optimization; cache partitioning eliminates cross-site caching benefit
- **Self-hosted static fonts**: Each weight requires separate file (150-400KB total), still requires HTTP requests
- **Variable fonts only**: Efficient (75-120KB), but still slower than zero-download system fonts

### Implementation:

```css
/* Modern System Font Stack */
body {
  font-family:
    -apple-system,
    BlinkMacSystemFont,
    'Segoe UI',
    Roboto,
    'Helvetica Neue',
    Inter,
    Arial,
    sans-serif;
  font-size: 16px;
  line-height: 1.6;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

/* Optional: Enhance headings with self-hosted variable font */
h1, h2, h3 {
  font-family:
    'InterVariable',
    -apple-system,
    'Segoe UI',
    sans-serif;
  font-weight: 700;
}
```

**If adding optional heading font**:
```html
<link rel="preload" href="/fonts/inter-variable.woff2" as="font" type="font/woff2" crossorigin>

<style>
@font-face {
  font-family: 'InterVariable';
  src: url('/fonts/inter-variable.woff2') format('woff2-variations');
  font-weight: 100 900;
  font-display: swap;
}
</style>
```

**Performance metrics expected**:
- System fonts only: Lighthouse 95-100, LCP <1.2s, CLS 0
- System + heading font: Lighthouse 90-98, LCP <1.5s, CLS <0.05

**Recommendation**: Start with pure system fonts. Add one variable font for headings only if design review requires brand differentiation.

## Analytics Selection

### Decision: Cloudflare Web Analytics (Free)

### Rationale:
- **Zero cost**: Free tier, no payment required
- **Performance**: 4.31 KB script (small enough for <3 external scripts requirement)
- **Privacy-first**: Cookie-free, GDPR compliant, no cookie banner needed
- **Sufficient metrics**:
  - Page views (visitor count)
  - Time on page (measures engagement, target: >20 seconds per SC-009)
  - Bounce rate (measures interest, target: <70% per SC-010)
  - Device/browser breakdown
- **Limitation**: No goal conversion tracking on free tier (but can track via form service email volume)

### Alternatives Considered:
- **Plausible Analytics**: <1 KB script but $9/month (paid tier, exceeds zero-budget constraint)
- **Fathom Analytics**: 1.6 KB script but $14-15/month (paid tier)
- **Simple Analytics**: 3 KB script, $9/month (paid tier)
- **Google Analytics 4**: 45-65 KB script (too large, privacy concerns, requires cookie banner)
- **No analytics**: Cannot measure success criteria; form service dashboard only

### Implementation:

**Prerequisites**: Use Cloudflare for DNS/CDN (can be free tier)

```html
<!-- In <head> -->
<script defer src="https://static.cloudflareinsights.com/beacon.min.js" data-cf-beacon='{"token": "YOUR_TOKEN"}'></script>
```

**Get your token**:
1. Add your domain to Cloudflare (free tier available)
2. Go to Analytics → Web Analytics
3. Copy the script with your token
4. Paste into `index.html` head

**Metrics tracked**:
- Unique visitors
- Page views
- Time on page
- Bounce rate
- Device type and browser
- Top pages, top referrers

**Limitation**: No form conversion goal tracking (alternative: monitor Web3Forms email count as proxy for conversion rate)

**External script count**: 2 total (Cloudflare Analytics + Web3Forms) - compliant with <3 scripts limit
